1. Agent Izolasyonu

Her Agent Kendi Workspace'inde Çalışsın

OpenClaw'da her agent kendi workspace dizinine sahip olmalıdır. Bu, dosya sistemi izolasyonu sağlar.

Nasıl Yapılır?

openclaw.json:
{
  "agents": {
    "list": [
      {
        "id": "securebot",
        "name": "🛡️ Security Agent",
        "workspace": "/home/openclaw/.openclaw/workspace-securebot",
        "agentDir": "/home/openclaw/.openclaw/agents/securebot/agent",
        "model": "stepfun/step-3.5-flash:free"
      },
      {
        "id": "main",
        "name": "🦀 Claw",
        "workspace": "/home/openclaw/.openclaw/workspace",
        "agentDir": "/home/openclaw/.openclaw/agents/main/agent",
        "model": "openrouter/auto"
      }
    ]
  }
}

Faydaları

  • ✅ Agent'lar birbirinin dosyalarını okuyamaz
  • ✅ Workspace corruption'ı tecrit eder
  • ✅ Agent-specific config/credentials izolasyonu
  • ✅ Backup/restore kolaylığı (her agent ayrı)

İleri Seviye: Docker Container Izolasyonu

Agent'ları tamamen ayrı container'larda çalıştırmak mümkündür:

{
  "agents": {
    "list": [
      {
        "id": "securebot",
        "sandbox": {
          "mode": "docker",
          "image": "openclaw-agent-base:latest",
          "memory": "256m",
          "cpu": "0.5",
          "network": "none"
        }
      }
    ]
  }
}

Bu, process-level izolasyon ekler. Agent çökse bile host etkilenmez.

2. Agent Communication

Agent'lar Birbirine Mesaj Atabilmeli (sessions_send)

OpenClaw built-in sessions_send aracıyla agent'lar arası mesajlaşma mümkündür.

Nasıl Kullanılır?

CLI'den:

openclaw sessions_send --to securebot -m "healthcheck"
openclaw sessions_send --to main -m "Durum raporu ver"

Python API:

from openclaw import sessions_send
response = sessions_send(agent="securebot", message="self-improve")
print(response)

Telegram'dan:

/session send securebot healthcheck
/session send main status

Güvenlik Kontrolleri

Sadece authorized agent'lar sessions_send kullanabilmeli:

{
  "tools": {
    "deny": ["sessions_spawn", "sessions_send"],
    "allow": ["sessions_send"]  // Sadece main agent'a izin ver
  }
}

Faydaları

  • ✅ Agent'lar arası koordinasyon (örneğin, Claw → Securi'ye rapor iste)
  • ✅ Task delegation (main agent, diğerlerine işi dağıtır)
  • ✅ Microservices mimarisi benzeri

3. Skill Sandboxing

Skill'ler Kendi Container'larında Çalışsın

Skill'ler doğrudan Gateway process'i içinde çalışır. Bu, bir skill hatasının tüm sistemi etkilemesine yol açabilir. Sandboxing, skill'leri ayrı container'larda (Docker) çalıştırarak izolasyon sağlar.

Nasıl Yapılır?

OpenClaw config'inde agent seviyesinde sandbox ayarı:

{
  "agents": {
    "list": [
      {
        "id": "securebot",
        "sandbox": {
          "mode": "docker",
          "image": "openclaw-skill-base:latest",
          "memory": "256m",
          "cpu": "0.5",
          "network": "none",
          "readOnlyFilesystem": true,
          "tmpfs": "/tmp:size=64m",
          "capDrop": ["ALL"],
          "securityOpt": ["no-new-privileges"]
        }
      }
    ]
  }
}

Docker Base Image Oluşturma

# Dockerfile
FROM node:18-alpine
RUN apk add --no-cache python3 make g++
WORKDIR /skill
COPY skill.json .
COPY lib/ lib/
RUN npm install --production
CMD ["node", "index.js"]

docker build -t openclaw-skill-base .

Faydaları

  • ✅ Skill çökmesi Gateway'ı etkilemez
  • ✅ Malicious skill host'a zarar veremez
  • ✅ Resource limit (memory, CPU)
  • ✅ Network isolation (exfiltration engellenir)
  • ✅ Seccomp/AppArmor profili uygulanabilir

Güvenlik Notları

  • Skill container'ı read-only mount edilmeli
  • capDrop: ["ALL"] ile tüm Linux capability'ler kaldırılmalı
  • network: none veya sadece whitelist
  • Container logları docker logs ile takip edilmeli

4. Skill Auto-Update

Güvenlik Güncellemeleri Otomatik Yüklensin

Skill'ler güvenlik açıklarına karşı duyarlıdır. Manuel update yapmak riskli olabilir. Otomatik update sistemi kurulumu:

Nasıl Yapılır?

1. Skill Git Repository'si

Skill'i private git repo'ya taşı (GitHub/GitLab).

git init
git add .
git commit -m "Initial"
git push origin main

2. Update Script

#!/bin/bash
# /home/openclaw/.openclaw/scripts/skill-update-securi.sh

SKILL_DIR="/home/openclaw/.openclaw/workspace/skills/securi-agent"
cd "$SKILL_DIR"

# Remote SHA
git fetch origin
REMOTE_SHA=$(git rev-parse origin/main)
LOCAL_SHA=$(git rev-parse HEAD)

if [ "$LOCAL_SHA" != "$REMOTE_SHA" ]; then
  echo "New version detected. Updating..."
  
  # Pull
  git pull origin main
  
  # Checksum verify (package.json version)
  # Eğer version değiştiyse devam et
  if git diff HEAD@{1} HEAD -- package.json | grep -q '"version"'; then
    echo "Version changed. Rebuilding..."
    npm install --production
    
    # Optional: GPG signature verify
    # gpg --verify signature.asc
    
    # Gateway restart (skill yüklenir)
    openclaw gateway restart
    
    # Telegram notification
    curl -X POST "https://api.telegram.org/bot$BOT_TOKEN/sendMessage" \
      -d "chat_id=$CHAT_ID" \
      -d "text=🛡️ Security Agent skill güncellendi (version: $(jq -r .version package.json))"
  else
    echo "No version change. Skipping."
  fi
fi

3. Cron Job (Günde 1 kez çalış)

0 2 * * * /home/openclaw/.openclaw/scripts/skill-update-securi.sh >> /var/log/securi-skill-update.log 2>&1

4. Onay Mekanizması (Manuel)

Eğer otomatik install istemezseniz, script size Telegram'dan onay isteyebilir:

curl -X POST "https://api.telegram.org/bot$BOT_TOKEN/sendMessage" \
  -d "chat_id=$CHAT_ID" \
  -d "text=🛡️ Security Agent yeni version (v1.2.0) mevcut. Yüklemek için /approve_update yazın." \
  -d "reply_markup={\"inline_keyboard\":[[{\"text\":\"Evet\",\"callback_data\":\"update_approve\"},{\"text\":\"Hayır\",\"callback_data\":\"update_deny\"}]]}"

Faydaları

  • ✅ Güvenlik açıkları hızla patch'lenir
  • ✅ Yeni feature'lar otomatik eklenir
  • ✅ Manual intervention minimize edilir
  • ✅ Version traceability (git log)

Riskler

  • ⚠️ Yeni version bug içerebilir → onay mekanizması önerilir
  • ⚠️ Breaking changes → test environment'de önce denemek gerek

def learn_from_scan(scan_results):
    new_rules = []
    for finding in scan_results:
        # Eğer bulgu mevcut rule'larda yoksa yeni rule oluştur
        if not rule_exists(finding):
            rule = {
                "id": generate_id(finding),
                "pattern": finding['pattern'],
                "severity": finding['severity'],
                "description": finding['description'],
                "remediation": suggest_remediation(finding)
            }
            new_rules.append(rule)

    # Git commit (versioning)

6. Benzer Sistemi Tanı (Anonymized Telemetry)

Başka Sunucularda Ne Tür Açıklar Var?

Securi, kendi tarama sonuçlarını (anonimleştirilmiş) diğer sistemlerle paylaşılarak kolektif bilgi edinebilir.

Nasıl Yapılır?

1. Telemetry Collection

def collect_telemetry():
    findings = run_healthcheck()
    anonymized = {
        "os": "Ubuntu 22.04",
        "openclaw_version": "2026.2.26",
        "findings": [
            {
                "cve": finding['cve'],
                "package": finding['package'],
                "severity": finding['severity'],
                "config_check": finding['config_check']
            } for finding in findings
        ],
        "timestamp": now(),
        "instance_id": hashlib.sha256(hostname + mac_address).hexdigest()[:16]
    }
    return anonymized

POST https://telemetry.openclaw.ai/v1/finding

7. Proaktif Önlemler

Vulnerability Prediction

Paket versiyonlarına göre risk skoru hesapla. CVSS database'sini kullan.

import requests

def predict_vuln(package, version):
    # NVD API'den CVE'leri çek
    url = f"https://services.nvd.nist.gov/rest/json/cves/2.0?cpeName=cpe:2.3:a:{package}:{version}"
    r = requests.get(url)
    cves = r.json()['vulnerabilities']
    
    # CVSS skorlarına göre risk hesapla
    risk_score = sum(cve['cvssMetrics'] for cve in cves)
    return risk_score

Configuration Drift

SSH config dosyası değişmiş mi? Bildir!

import hashlib

def check_drift(file_path, known_hash):
    current_hash = hashlib.sha256(open(file_path, 'rb').read()).hexdigest()
    if current_hash != known_hash:
        alert(f"Configuration drift detected: {file_path}")
